🏗️ Mikrozerbitzuak eta Arkitektura Modernoa

Eduki interaktibo honetan sistema modernoen arkitektura aztertuko dugu: monolitoa, mikrozerbitzuak, API Gateway, Docker eta cloud teknologien harremana.

Helburua ez da galderei erantzutea, baizik eta ulertzea nola funtzionatzen duen sistema oso batek barrutik.

🏗️ Zer da arkitektura bat?

🔍 Sakatu definizioa ikusteko
Arkitektura batek definitzen du:
  • Sistema nola antolatzen den
  • Osagaien arteko komunikazioa
  • Datuen kudeaketa
  • Eskalatzea eta hedapena

🧱 Aplikazio monolitikoa

Sistema osoa unitate bakarrean dago: logika, datuak eta interfazea batera.

⚠️ Egoera bat ikusi
Imajinatu online denda bat:

Erabiltzaileak, ordainketak eta produktuak sistema berean daude.

👉 Ordainketa moduluak huts egiten badu, aplikazio osoa kaltetu daiteke.
💭 Hausnarketa
Monolitoak sinpleak dira hasieran, baina sistema hazten denean, aldaketak zailagoak eta arriskutsuagoak bihurtzen dira.

🧩 Mikrozerbitzuen ideia

Sistema txiki eta independenteetan banatzen da.

💭 Zergatik banatu?
Zerbitzu bakoitzak ardura bakarra du. Horrek malgutasuna eta eskalagarritasuna handitzen ditu, baina konplexutasuna ere bai.

🔗 Zerbitzuen arteko komunikazioa

Frontend → API Gateway → Mikrozerbitzuak
📡 Egoera erreala
Erabiltzaile batek eskaera bat egiten du. API Gateway-k eskaera jasotzen du eta Payments zerbitzura bidaltzen du. Ondoren Notifications zerbitzuak erabiltzaileari jakinarazpena bidaltzen dio.
💭 Zer gertatzen da zerbitzu bat huts egiten badu?
Sistema osoa ez da derrigorrez erortzen, baina funtzionalitate batzuk galtzen dira. Honek tolerantzia eta resilentzia beharra erakusten du.

⚙️ APIak, Docker eta Cloud

🔌 API kontzeptua
APIek sistema desberdinak komunikatzea ahalbidetzen dute. JSON da ohiko datu formatua.
🐳 Docker
Zerbitzu bakoitza edukiontzi batean exekutatzen da, ingurune desberdinetan berdin funtzionatzeko.
☁️ Cloud (EC2)
Aplikazioak internet bidez hedatzeko aukera ematen du, eskalagarritasuna eta erabilgarritasuna handituz.

⚖️ Monolitoa vs Mikrozerbitzuak

🧱 Monolitoa
Sinplea hasieran, baina haztean zaila mantentzen. Sistema osoa batera hedatzen da.
🧩 Mikrozerbitzuak
Banatutako sistema. Malgutasun handiagoa, baina konplexutasun tekniko handiagoa.
💭 Hausnarketa orokorra
Ez dago arkitektura “hoberik”. Testuinguruaren arabera aukeratzen da. Sistema txikietan monolitoa egokiagoa izan daiteke, sistema handietan mikrozerbitzuak.

🎉 Ondorioa

Mikrozerbitzuak ez dira bakarrik teknologia bat, baizik eta sistema konplexuak antolatzeko filosofia bat.

  • ✔ Banaketa logikoa
  • ✔ Eskalagarritasuna
  • ✔ Komunikazio sistema
  • ✔ Konplexutasun kudeatua
